TagMemoRAG 官方文档

TagMemoRAG 是一个面向真实文档的浏览器优先 RAG 系统。它帮助团队把 PDF、DOCX、Markdown、文本和可选 OCR 内容变成可验证、有引用来源的问答体验。

开始阅读 查看仓库

快速开始

正常用户不需要理解内部索引细节。推荐先按照浏览器流程完成一次真实体验。

  1. 打开 Manual Library创建或加载知识库，上传第一份手册并填写产品元数据。
  2. 重建索引执行 full 或 incremental rebuild，让文档进入可检索状态。
  3. 检查 Readiness确认文档、检索、来源预览、OCR 和交付检查是否就绪。
  4. 进入 Q&A用真实用户问题测试回答，并检查引用来源是否可靠。

核心概念

知识库

一组可检索文档、索引、元数据和运行状态的边界。

手册

具有产品、型号、语言、标签和来源文件信息的文档单元。

引用

回答中的证据指针，用户可以通过来源卡片核对。

Readiness

面向交付的检查清单，帮助判断是否能给用户试用。

手册库

Manual Library 是管理员处理文档的主入口。它负责上传、校验、标签治理、脏状态检查、诊断和重建。

第一次使用 先上传一份小而真实的文档，完成一次重建，再去 Q&A 问一个真实问题。

用户问答

Q&A 页面面向最终用户，不暴露调参、节点 ID、plan ID 或底层诊断字段。用户看到的是问题、回答、引用和安全的来源说明。

用户问题 → 路由知识库 → 检索证据 → 生成回答 → 展示引用来源

RAG Readiness

Readiness 页面把“能不能交付给用户体验”拆成可扫描的检查项：知识库是否加载、手册是否可检索、评测是否通过、Q&A 是否可用、OCR 和来源预览是否清楚。

质量治理

Retrieval Quality 和 eval report 用来持续观察弱问题、期望证据、反馈闭环和回归风险。发布前建议至少跑一次非性能单元/e2e 测试和 baseline eval。

OCR 的作用

OCR 处理扫描版 PDF 中没有原生文本的页面。它不是单独的问答通道，而是把识别出的文本送入同一套解析、分块、检索和回答流程。

注意 OCR 需要操作者安装本地 Tesseract 工具，并在配置中显式启用。

人员与权限

建议按最小权限分配：普通问答用户只需要 search，负责文档维护的人员需要 rebuild，管理员才需要管理密钥和访问边界。

部署与发布

当前公开文档通过 GitHub Pages 发布。项目本体可以自托管运行，发布前应确认本地质量门、GitHub Actions、Readiness 页面和真实浏览器流程。

FAQ

不安装项目可以看这个文档站吗？

可以。这个站点是静态页面，通过 GitHub Pages 公开访问，不需要运行 TagMemoRAG。

Q&A 页面和管理员 Workbench 是一回事吗？

不是。Q&A 面向最终用户，Workbench 和其他 admin 页面用于调试、上传、重建和治理。

回答可信度怎么判断？

先看回答是否引用了正确来源，再通过来源卡片核对手册片段、页码、OCR 标记和来源预览。
