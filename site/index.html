Public release v0.1.0

面向真实文档的浏览器优先 RAG

TagMemoRAG 把产品手册、团队文档、PDF、DOCX、文本和可选 OCR 内容转成可验证、有引用来源的问答体验。默认本地 demo 不需要外部 API key，适合快速试用；生产配置可接入 Qdrant、S3 兼容对象存储、HTTP embedding、reranker 和 answer provider。

Start

5 分钟浏览器体验

从仓库根目录运行本地 demo，然后打开浏览器页面体验完整 RAG 流程。

  1. 安装依赖uv sync --extra dev
  2. 生成本地 demo KBuv run python -m tagmemorag demo library-qa --config examples/config/qa-demo.yaml --clean
  3. 启动服务uv run python -m tagmemorag serve --host 127.0.0.1 --port 8000 --config examples/config/qa-demo.yaml
  4. 打开浏览器http://127.0.0.1:8000/qa?kb_name=default

Use

浏览器使用路径

Manual Library

上传文档、填写元数据、检查 searchable 状态、触发 rebuild，并跳转到 Q&A。

Ask Q&A

面向最终用户的问答页。空知识库会引导上传，答案会展示引用和来源卡。

RAG Readiness

检查知识库、文档、检索、Q&A、OCR、来源预览和交付状态。

Retrieval Quality

审核反馈、记录期望证据、生成 eval 样本，降低回归风险。

Documents

真实文档支持矩阵

格式当前支持说明
Markdown / TXT直接支持适合结构化说明、FAQ、操作 SOP。
文本 PDF直接支持保留页码元数据，可生成来源预览。
DOCX支持 readable DOCX通过 OpenXML 文本抽取并物化 Markdown，同时保留原始 DOCX 来源信息。
扫描 PDF需要 OCR 配置安装 Tesseract 和 PDF renderer 后可把识别文本送入同一套 RAG 流程。
Legacy DOC暂不支持二进制 Word 需要单独依赖和安全边界决策。
真实样本验证 仓库内保留 washer、oven、refrigerator、dryer 等真实产品手册样本；浏览器黑盒测试覆盖多类真实 PDF 的上传、索引、问答和来源预览。
Trust

来源核验

Q&A 页面不把底层调参、节点 ID、存储 key 或 blob key 暴露给普通用户。用户看到的是回答、引用 chip、来源卡、页码、DOCX/OCR 标记和安全的预览链接。

用户问题 → 检索证据 → 生成回答 → 引用 chip → 来源卡 → 安全预览

Operate

生产部署体验

TagMemoRAG 当前适合单机或单 writer 的生产化试点。部署时要把配置验证、provider probe、备份恢复、权限、监控和回滚放进同一条 checklist。

备份恢复覆盖 graph artifacts、QueryPlan SQLite、manual registry、blob store、Qdrant collection、config 和 secrets。
权限普通用户只给 search；文档维护者给 rebuild；admin 留给密钥、删除和审核操作。
监控至少检查 /health、/ready、/metrics、dirty state、registry/blob verify 和 retained pilot report。
回滚坏配置、失败 rebuild、Qdrant/S3 outage、坏 bundle import 都有独立 playbook。

Release

发布检查清单

  1. Preflight配置验证、本地 smoke、质量门、版本和 commit 记录。
  2. Backup导出 bundle，确认 SQLite、S3、Qdrant 和 config/secrets 的恢复路径。
  3. Access启用 auth，按最小权限分配 search、rebuild、admin。
  4. Browser acceptance上传真实文档，问答，检查引用和来源预览。
  5. Rollback确认配置、rebuild、Qdrant、S3/blob 和 bundle import 的回滚动作。

FAQ

常见问题

不安装项目可以看这个文档站吗？

可以。这个站点是静态页面，通过 GitHub Pages 公开访问，不需要运行 TagMemoRAG。

Q&A 页面和管理员 Workbench 是一回事吗？

不是。Q&A 面向最终用户，Workbench 和其他 admin 页面用于调试、上传、重建和治理。

回答可信度怎么判断？

先看回答是否引用了正确来源，再通过来源卡片核对手册片段、页码、OCR 标记和来源预览。
