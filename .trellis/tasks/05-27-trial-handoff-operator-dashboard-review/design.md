# Trial Handoff And Operator Dashboard Review Design

## Scope

This child task is documentation and operator-flow review only. It does not modify product runtime behavior.

## Deliverable

Add a current handoff document under `docs/` that supersedes older post-merge handoff notes for the current master state. The document should serve as the operator's starting page for a local trial.

## Content Shape

The handoff should include:

- local offline startup path;
- browser dashboard map;
- retained pilot/browser report commands;
- feedback triage loop;
- upload/rebuild recovery pointers;
- auth and live-provider boundaries;
- links to deeper runbooks.

## Compatibility

Existing docs should remain valid. Add links rather than deleting historical handoff notes.
