# Design

## Scope

Extend the existing general-web eval lane rather than creating a new framework. The flow remains:

1. `scripts/seed_general_web_eval.sh` fetches selected public URLs with `tagmemorag knowledge sample-web`.
2. The importer materializes Markdown and `.metadata.json` sidecars under `.tmp/general-web-eval/general_web/public_web/`.
3. `tagmemorag eval run` validates retrieval against `tests/fixtures/eval/general_web.jsonl`.
4. `scripts/diag_general_web_answer_eval.py` validates generated extractive answers from live retrieval.
5. `scripts/diag_mixed_domain_eval.py --stage-from-defaults` stages real manuals plus the seeded public docs into one KB.

## Source Selection

Use official or canonical documentation/help pages with relatively stable prose:

- Python Tutorial (`software_docs`, `documentation`, tags `software-docs`, `python`)
- GitHub Hello World (`software_docs`, `documentation`, tags `software-docs`, `github`)
- MDN HTTP caching (`web_platform_docs`, `documentation`, tags `web-platform`, `http`, `caching`)
- Google Search Help: Learn search tips (`help_center`, `support_article`, tags `help-center`, `search`)

The seed script should call `knowledge sample-web` once per domain/type group so sidecar metadata can carry useful domain/doc_type/tags.

## Eval Contracts

Retrieval eval cases should assert:

- `source_file` generated by the public web importer.
- `text_contains` strings observed from the public page and likely preserved by the importer.
- Metadata fields `domain`, `doc_type`, and `manual_id` where practical.
- Optional `top_k_override` for long pages where relevant text may be split among multiple chunks.

Answer diagnostic remains schema-compatible. It treats all general-web cases as expected grounded/relevant/citation-supported. If a newly added case is too broad for deterministic noop answer generation, narrow the query/expected text rather than adding a special evaluator.

## Compatibility

- No fetched content is committed; the corpus remains materialized under `.tmp`.
- Existing Python/GitHub cases retain ids and semantics.
- Mixed-domain diagnostic automatically picks up the expanded public-web corpus via `--stage-from-defaults`.
- The CI fixture-only baseline remains unchanged because `general_web.jsonl` is already excluded from default fixture-only CI.

## Rollback

If a source page changes enough to break deterministic expectations, revert that source and its cases independently. The existing Python/GitHub seed remains the minimum viable general-web slice.
