## 2. Core Contracts

### 2.1 `AgentTool` & `ToolObservation` (`tools/base.py`)

```python
from typing import Protocol, runtime_checkable

@dataclass(frozen=True)
class ToolObservation:
    """Output of a tool call. Serialized into plan_steps.observation_json."""
    payload: dict[str, Any]              # tool-specific structured data
    tokens_consumed: int = 0
    latency_ms: int = 0
    warnings: tuple[str, ...] = ()
    rerank_result: "RerankResult | None" = None  # carried for grader; not serialized

@dataclass(frozen=True)
class AgentStepCtx:
    plan: QueryPlan
    guard: BudgetGuard
    settings: "Settings"
    step_idx: int
    history: tuple["StepRecord", ...]    # immutable view of prior steps
    state_dir: str                       # for any tool needing per-kb assets

@runtime_checkable
class AgentTool(Protocol):
    name: str                            # "retrieve" | "grade" | "rewrite" | "final" (extensible)
    description: str
    input_schema: dict                   # JSON Schema, OpenAI tool-call compatible

    def __call__(self, args: dict[str, Any], ctx: AgentStepCtx) -> ToolObservation: ...
```

### 2.2 `AgentToolRegistry` (`tools/registry.py`)

```python
class AgentToolRegistry:
    def register(self, tool: AgentTool) -> None: ...
    def get(self, name: str) -> AgentTool: ...
    def has(self, name: str) -> bool: ...
    def names(self) -> tuple[str, ...]: ...
    def openai_schemas(self) -> list[dict]:
        """List of OpenAI 'tools' entries for decision LLM."""
        return [{"type": "function",
                 "function": {"name": t.name,
                              "description": t.description,
                              "parameters": t.input_schema}}
                for t in self._tools.values()]

def default_registry(deps: ToolDeps) -> AgentToolRegistry:
    """Factory used by tests and (eventually, in C6) by the API layer."""
    reg = AgentToolRegistry()
    reg.register(RetrieveTool(deps.search_runtime))
    reg.register(GradeTool(deps.dispatcher))
    reg.register(RewriteTool())
    reg.register(FinalTool(deps.answer_generator))
    return reg
```

### 2.3 `AgentState`, `StepRecord`, `GradeOutcome` (`state.py`)

```python
@dataclass(frozen=True)
class GradeOutcome:
    top1_score: float
    margin: float
    depth: int
    signal: Literal["high", "low", "inconclusive", "no_signal"]
    reason: str

@dataclass
class StepRecord:
    step_idx: int
    tool: str                            # registry key
    args: dict[str, Any]
    observation: ToolObservation
    grade: GradeOutcome | None           # None for non-grade steps
    decision_source: Literal["rule", "llm"]
    rationale: str
    ts: str                              # ISO-8601 UTC

@dataclass
class AgentState:
    plan: QueryPlan
    history: list[StepRecord] = field(default_factory=list)
    last_retrieve_obs: ToolObservation | None = None
    last_grade: GradeOutcome | None = None
    classic_fallback_answer: AnswerGeneration | None = None
    final_answer: AnswerGeneration | None = None

    def append(self, rec: StepRecord) -> None: ...
    def can_iterate(self, guard: BudgetGuard) -> bool: ...
    def finalize(self) -> AnswerGeneration: ...
```

### 2.4 `Budget` extension (`queryplan/plan.py`)

```python
@dataclass(frozen=True)
class Budget:
    # existing fields preserved verbatim
    ...
    # new optional agentic fields (default values match parent design §3.1)
    max_iterations: int = 3
    max_agent_tokens: int = 4096
    max_tool_calls: int = 12

    def to_dict(self) -> dict[str, Any]:
        d = {... existing fields ...}
        # only emit agentic keys when caller actually overrode them,
        # so existing classic fixtures stay byte-equivalent.
        if self.max_iterations != 3 or self.max_agent_tokens != 4096 \
           or self.max_tool_calls != 12:
            d["max_iterations"] = self.max_iterations
            d["max_agent_tokens"] = self.max_agent_tokens
            d["max_tool_calls"] = self.max_tool_calls
        return d
```

`BudgetGuard` gains:

```python
def iterations_left(self) -> int: ...
def tokens_left(self) -> int: ...
def tool_calls_left(self) -> int: ...

def consume_tool_call(self) -> None: ...
```

### 2.6 Driver loop (`driver.py`)

```python
def run_agent(
    *,
    plan: QueryPlan,
    registry: AgentToolRegistry,
    guard: BudgetGuard,
    decision_gen: DecisionGenerator,
    settings: "Settings",
    plan_log: PlanLog,
) -> AnswerGeneration:
    state = AgentState(plan=plan)
    while True:
        step_idx = len(state.history)
        # 1. Decide next action
        action, source, rationale = _decide_next(
            state=state, registry=registry, decision_gen=decision_gen
        )
        # 2. Budget check before tool call
        if guard.tool_calls_left() <= 0 or guard.iterations_left() <= 0:
            action, source, rationale = _force_final(state, "budget")
        # 3. Execute tool
        ctx = AgentStepCtx(plan=plan, guard=guard, settings=settings,
                           step_idx=step_idx, history=tuple(state.history),
                           state_dir=...)
        obs = registry.get(action.name)(action.args, ctx)
        guard.consume_tool_call()
        guard.consume_tokens(obs.tokens_consumed)
        # 4. Materialize step record
        rec = _build_step_record(step_idx, action, obs, source, rationale,
                                 grade=_extract_grade(obs))
        state.append(rec)
        plan_log.append_step_async(plan.plan_id, rec)
        # 5. Termination
        if action.name == "final":
            return state.finalize()
        guard.consume_iteration()
        if guard.iterations_left() <= 0:
            return _terminate_with_classic_fallback(state, registry, ctx)

def _decide_next(state, registry, decision_gen) -> tuple[Action, str, str]:
    """Apply rule fastpath table from parent design §3.7.

    In C1 this resolves to:
      - step 0 → ("retrieve", default_args, "rule", "step0")
      - else → ("grade", default_args, "rule", "post_retrieve")
      - signal == "no_signal" → ("final", default_args, "rule",
                                  "no_signal_classic_fallback")

    Because the C1 GradeTool stub always emits no_signal, every run
    walks: retrieve → grade → final. decision_gen is therefore never
    consulted in C1 except via a unit test that exercises the
    inconclusive branch with a synthetic GradeOutcome.
    """
```

### 2.7 Replay branch (`replay/runner.py` + `agentic/replay.py`)

```python
# in replay/runner.py replay_plan(...)
if plan_log.has_steps(plan.plan_id):
    return agentic.replay.replay_steps(
        plan=plan, settings=settings, embedder=embedder, state=graph_state,

# in agentic/replay.py
def replay_steps(...) -> ReplayCaseResult:
    """Walk stored StepRecords, executing each tool with stored args.

    Per D7:
    - Never re-prompt the decision LLM.
    - Compare `tool` / `signal` / `decision_source` exactly (MUST).
    - Compare `args` key-set only.
    - Compare evidence with `_jaccard` (reused from replay/metrics.py).
    Returns a ReplayCaseResult shaped like classic so existing
    aggregation works; emits StepReplayVerdict / AgentRunReplayVerdict
    via plan_log side-channel for debug surfaces.
    """
```

In C1, every step has `decision_source == "rule"` and stub tools are
deterministic, so verdicts must always be `match` — that is AC1.4's
formal claim.

### 2.8 D6 invariant test

`tests/unit/test_reranker_dispatcher_cache_key_invariant.py`:

```python
def test_cache_key_independent_of_step_idx(monkeypatch):
    """Lock D6: dispatcher cache key uses (plan, candidates) only.

    Adding agent step_idx (or any state-dependent field) to the key
    would break agentic correctness invisibly."""
    plan = _make_plan("kb")
    cands_A = [_cand("c1"), _cand("c2")]
    cands_B = [_cand("c1"), _cand("c3")]
    dispatcher = _build_dispatcher()

    key_A = dispatcher._cache_key(plan, cands_A)
    key_B = dispatcher._cache_key(plan, cands_B)
    key_A_again = dispatcher._cache_key(plan, cands_A)

    assert key_A != key_B, "different candidates must produce different keys"
    assert key_A == key_A_again, "same (plan, candidates) must produce same key"

    # Negative regression: any future commit that adds a step-aware
    # parameter MUST break this test by either changing the signature
    # or producing a different key. Comment links D6 in PRD.
```

## 3. Compatibility & Rollback

- Classic eval fixtures must produce byte-identical answers: enforced
  by AC1.1 (eval-runner diff against `main`). The only realistic risk
  is the `Budget.to_dict` change — mitigated by emitting agentic keys
  conditionally so existing fixtures don't gain new JSON keys.
- `plan_steps` is purely additive. Rolling back means dropping the
  table; no migration.
- `replay/runner.py` change is a single `if has_steps` branch; classic
  plans never trigger it.
- The four new fixture files only enter regression once their baselines
  exist. Rolling back deletes them.

## 4. Trade-offs

- **Decision generator stubbed, not implemented**: keeps C1 small and
  defers LLM provider decisions to C6. Cost: one `NotImplementedError`
  raise path that must be unreachable in C1 tests.
- **GradeTool calls dispatcher despite returning no_signal stub**: pays
  one rerank call per agentic stub run (which is exactly what classic
  pays anyway), in exchange for the D6 invariant test having a real
  caller and the agentic-stub path running through the same scoring
  fabric as classic.
- **Budget extension uses defaults instead of None sentinels**: simpler
  code, but requires conditional serialization to keep classic JSON
  fixtures byte-equivalent. Verified by AC1.1.

## 6. Out-of-Scope Reaffirmations

C1 explicitly does not implement:

- LLM-driven decision generation (only stub `RuleOnlyDecisionGenerator`).
- Real signal computation (GradeTool stub forces `no_signal`).
- Real query rewriting (RewriteTool is identity).
- Adaptive routing (no flavor-A logic in driver — the rule fastpath
  table is hard-coded, not configurable per flavor).
- Budget breach event surface (`agentic.budget_exhausted` emission is
  C5).
- API / CLI / config surface (C6).
- Provider verification decision step (C6 / D5).
