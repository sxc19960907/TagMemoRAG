## 2. 文件名 → category 推断（D3 best-guess）

| Pattern | Category | Brand | 备注 |
|---|---|---|---|
| `ASKO W*` | `washer` | ASKO | W 前缀通常是洗衣机 |
| `HISENSE BSA*` | `dishwasher` | HISENSE | BSA = Built-in Slim/Standard ? best-guess，跑通后用户校正 |
| `HISENSE DH*` (DHGA901NL / DHQE800BW2) | `dishwasher` | HISENSE | DH = DishWasher |
| `HISENSE HR*` (HR6FDFF701SW) | `refrigerator` | HISENSE | HR = home refrigerator |

**fallback**：扫 PDF 第一页前 1000 字符，若含"洗衣机/washer"→ washer，含"洗碗机/dishwasher"→ dishwasher，含"冰箱/refrigerator/fridge"→ refrigerator。
**保底**：写未识别到 stderr 提示用户手动改 sidecar，不阻塞流程。

## 4. fixture schema 适配 PDF parser 输出

PDF parser 产出的 chunk 是 `header="Page N"` —— `_matches` 要求 `header == expectation.header`。
两种处理：

- (A) **fixture 写 `header="Page 5"` 等具体页号** — 但答案在哪页是 build 后才知道，标注耦合度高
- (B) **fixture `header=""` (空)，让 matcher 跳过 header** — schema 友好

读 `_matches` 实现确认 (B) 行为：

```python
# src/tagmemorag/eval/matching.py:65
if expectation.header and result.header.strip() != expectation.header.strip():
    return False
```

`expectation.header` 是空串时跳过 — 走 `text_contains` + `source_file` 双约束即可。**走 (B)**。

## 5. 实验组合（D5）

| Config | spike_enabled | resonance | residuals | geodesic |
|---|---|---|---|---|
| `vec-only` | **false** | - | - | - |
| `wave-baseline` | true | false | false | false |
| `wave-residuals` | true | false | true | false |
| `wave-resonance` | true | true | false | false |

不测 `geodesic` / `all-on`：geodesic 在 readiness 实证已 KEEP_OFF；all-on 在 readiness 也 = only-geodesic。

每组都用 siliconflow Qwen-VL 4096 维。

## 6. 关键实证产出

输出 `research/realmanuals-eval-report.md` 必须含：

1. **数据规模**：KB build 后 chunk count / total tokens / build time
2. **绝对指标表**：4 config × N suite × 4 metric
3. **命题验证表**：`wave-baseline - vec-only` 的 delta（这是核心）
4. **flag 增量表**：`wave-residuals - wave-baseline` / `wave-resonance - wave-baseline`
5. **case 级别分析**：挑 3-5 个有趣 query，列 wave 和 vec-only 实际召回的 top-K，看看 wave 是不是真的"跨章节并列召回"得更全
6. **结论**：根据数据给"赢/平/输"判断 + 后续建议

## 7. 风险点

- **PDF parser 一页一 chunk 结构降级浪潮算法优势** — 已在 PRD 接受。如果实测 wave 没赢 KNN，第一嫌疑是这个，不是算法本身
- **API quota** — siliconflow Qwen-VL 4096 维 × 236 page × 4 config × 跨多次调用，估计 < 5000 次 API call，应该够用
- **fixture 标注偏差** — 用 dual embedder 候选 + 人工 review（autonomous mode 默认接受 AI 推荐），但仍有"AI 也漏的合理答案"风险。`--extra-candidates` 兜底机制已经在 relabel 工具里
- **实验时间** — 单次完整跑预计 30-60 分钟（5 PDF embed 是大头，跑一次复用即可）

## 8. Rollback

每个 stage 独立 commit；如某 stage 卡住可独立 revert。fixture jsonl + sidecar 都是新建文件，rm 即可。
