# CI actions runtime maintenance

## Goal

Remove the GitHub Actions Node.js 20 deprecation warning from project workflows while preserving the current Quality CI and public site publishing behavior.

## Requirements

- Update workflow actions that currently run on deprecated Node.js 20 runtimes when newer compatible major versions are available.
- Preserve Quality CI triggers for pull requests and pushes to `master` / `main`.
- Preserve public site publishing triggers and the `gh-pages` publishing behavior.
- Keep the workflows simple; do not introduce unrelated CI restructuring.
- Verify the relevant workflow YAML files parse locally where possible.
- Push and verify GitHub Actions after the change.

## Acceptance Criteria

- [x] `.github/workflows/quality.yml` no longer uses actions that emit the Node.js 20 deprecation warning.
- [x] `.github/workflows/pages.yml` no longer uses third-party actions that emit the Node.js 20 deprecation warning.
- [x] Local focused checks pass.
- [x] Remote Quality CI passes after push.
- [x] Public site publish workflow remains available and unchanged in behavior for site changes.

## Verification Notes

- `actions/checkout@v6` and `astral-sh/setup-uv@v8.1.0` both declare `runs.using: node24`.
- `peaceiris/actions-gh-pages@v4` declares `runs.using: node20`, so the public-site publisher was replaced with an equivalent shell-based orphan `gh-pages` publish step.
- Remote Quality CI passed for commit `f40987d` without the Node.js 20 annotation.
- Repository-managed `Publish Public Site` passed for commit `f40987d`.
- GitHub's automatic `pages-build-deployment` workflow still emits a Node.js 20 annotation from platform-managed `actions/checkout@v4` and `actions/upload-artifact@v4`. That workflow is generated by GitHub Pages after `gh-pages` updates and is outside this repository's workflow files.
